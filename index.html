Перелік основних команд:

  • git config - global user.name
  • git config - global user.email
  • git init - створення пустого репозиторію
  • git status - відображення статусу репозиторію
  • git add - додати нові або змінені файли, щоб наступним кроком використати git commit
  • git commit - це знімок внесених змін в репозиторій
  • git log - відображає історію комітів
  • git checkout - зміна гілки
  • git branch - відображення поточної гілки
  • git push - відправити зміни в репозиторій
  • git pull - отримання даних з віддаленого репозиторію
  • git merge - об'єднання змін в одну гілку
  • git clone - клонування проекту в нову створену директорію
  • .gitignore файл - це певні файли, які будуть ігноруватись гітом

Advanced:

  • git revert - це команда, яку ми використовуємо, коли хочемо взяти попередній commit та додати його як новий commit, зберігаючи logнедоторканим.
  • git reset - це команда, яку ми використовуємо, коли хочемо перемістити репозиторій назад до попереднього commit
  • git rebase - це процес переміщення послідовності стосовно нового базового commit або їх обєднання
  • git cherry-pick - дозволяє інтегрувати вибрані окремі коміти з будь-якої гілки в поточну гілку HEAD.
  • git stash - використовує змінений стан вашого робочого каталогу, змінені зареєстровані файли та проіндексовані зміни, і зберігає їх у сховищі незавершених змін, які можна в будь-який час змінити назад.
